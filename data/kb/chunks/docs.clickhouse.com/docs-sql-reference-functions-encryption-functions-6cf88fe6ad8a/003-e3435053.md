## decrypt[вҖӢ](#decrypt "Direct link to decrypt")

Introduced in: v20\.12\.0

This function decrypts an AES\-encrypted binary string using the following modes:

- aes\-128\-ecb, aes\-192\-ecb, aes\-256\-ecb
- aes\-128\-cbc, aes\-192\-cbc, aes\-256\-cbc
- aes\-128\-ofb, aes\-192\-ofb, aes\-256\-ofb
- aes\-128\-gcm, aes\-192\-gcm, aes\-256\-gcm
- aes\-128\-ctr, aes\-192\-ctr, aes\-256\-ctr
- aes\-128\-cfb, aes\-128\-cfb1, aes\-128\-cfb8

**Syntax**

```
decrypt(mode, ciphertext, key[, iv, aad])

```

**Arguments**

- `mode` вҖ” Decryption mode. [`String`](/docs/sql-reference/data-types/string)
- `ciphertext` вҖ” Encrypted text that should be decrypted. [`String`](/docs/sql-reference/data-types/string)
- `key` вҖ” Decryption key. [`String`](/docs/sql-reference/data-types/string)
- `iv` вҖ” Initialization vector. Required for `-gcm` modes, optional for others. [`String`](/docs/sql-reference/data-types/string)
- `aad` вҖ” Additional authenticated data. Won't decrypt if this value is incorrect. Works only in `-gcm` modes, for others throws an exception. [`String`](/docs/sql-reference/data-types/string)

**Returned value**

Returns decrypted plaintext. [`String`](/docs/sql-reference/data-types/string)

**Examples**

**Correctly decrypting encrypted data**

```
-- Re-using the table from the encrypt function example
SELECT comment, hex(secret) FROM encryption_test;
