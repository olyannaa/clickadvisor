An alias is a user\-defined name for an [expression](#expressions) in a query.

```
expr AS alias

```

The parts of the syntax above are explained below.

| Part of syntax Description Example Notes| `AS` The keyword for defining aliases. You can define the alias for a table name or a column name in a `SELECT` clause without using the `AS` keyword. `SELECT table_name_alias.column_name FROM table_name table_name_alias`. In the [CAST](/docs/sql-reference/functions/type-conversion-functions#CAST) function, the `AS` keyword has another meaning. See the description of the function.| `expr` Any expression supported by ClickHouse. `SELECT column_name * 2 AS double FROM some_table` | `alias` Name for `expr`. ### Notes on Usage[​](#notes-on-usage "Direct link to Notes on Usage")

- Aliases are global for a query or subquery, and you can define an alias in any part of a query for any expression. For example:

```
SELECT (1 AS n) + 2, n`.

```

- Aliases are not visible in subqueries and between subqueries. For example, while executing the following query ClickHouse generates the exception `Unknown identifier: num`:

```
`SELECT (SELECT sum(b.a) + num FROM b) - a.a AS num FROM a`

```

- If an alias is defined for the result columns in the `SELECT` clause of a subquery, these columns are visible in the outer query. For example:

```
SELECT n + m FROM (SELECT 1 AS n, 2 AS m)`.

```
