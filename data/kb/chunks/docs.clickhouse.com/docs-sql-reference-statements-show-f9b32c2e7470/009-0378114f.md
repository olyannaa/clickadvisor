`SHOW SETTINGS` statement returns a list of system settings and their values. It selects data from the [`system.settings`](/docs/operations/system-tables/settings) table. ### Syntax[вҖӢ](#syntax-21 "Direct link to Syntax") ``` SHOW [CHANGED] SETTINGS LIKE|ILIKE <name> ``` ### Clauses[вҖӢ](#clauses "Direct link to Clauses")

`LIKE|ILIKE` allow to specify a matching pattern for the setting name. It can contain globs such as `%` or `_`. `LIKE` clause is case\-sensitive, `ILIKE` вҖ” case insensitive.

When the `CHANGED` clause is used, the query returns only settings changed from their default values.

### Examples[вҖӢ](#examples-6 "Direct link to Examples")

Query with the `LIKE` clause:

```
SHOW SETTINGS LIKE 'send_timeout';

```

## SHOW ENGINES[вҖӢ](#show-engines "Direct link to SHOW ENGINES")

The `SHOW ENGINES` statement outputs the content of the [`system.table_engines`](/docs/operations/system-tables/table_engines) table,
that contains description of table engines supported by server and their feature support information.

### Syntax[вҖӢ](#syntax-23 "Direct link to Syntax")

```
SHOW ENGINES [INTO OUTFILE filename] [FORMAT format]

```

## SHOW FUNCTIONS[вҖӢ](#show-functions "Direct link to SHOW FUNCTIONS")

The `SHOW FUNCTIONS` statement outputs the content of the [`system.functions`](/docs/operations/system-tables/functions) table.

### Syntax[вҖӢ](#syntax-24 "Direct link to Syntax")

```
SHOW FUNCTIONS [LIKE | ILIKE '<pattern>']

```

If either `LIKE` or `ILIKE` clause is specified, the query returns a list of system functions whose names match the provided `<pattern>`.
