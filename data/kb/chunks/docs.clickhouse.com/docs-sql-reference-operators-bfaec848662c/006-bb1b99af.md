The result is a [DateTime](/docs/sql-reference/data-types/datetime) or [DateTime64](/docs/sql-reference/data-types/datetime64) representing the date at the given time of day. The operation is commutative. The result type depends on the operand types:

NoteThe result uses the [session timezone](/docs/operations/settings/settings#session_timezone) (or server default timezone if no session timezone is set). The [`date_time_overflow_behavior`](/docs/operations/settings/formats#date_time_overflow_behavior) setting controls what happens when the result is outside the representable range.

Examples:

```
SET use_legacy_to_time = 0;
SELECT toDate('2024-07-15') + toTime('14:30:25') AS dt, toTypeName(dt);
```

## Logical AND Operator[вҖӢ](#logical-and-operator "Direct link to Logical AND Operator")

Syntax `SELECT a AND b` вҖ” calculates logical conjunction of `a` and `b` with the function [and](/docs/sql-reference/functions/logical-functions#and).

## Logical OR Operator[вҖӢ](#logical-or-operator "Direct link to Logical OR Operator")

Syntax `SELECT a OR b` вҖ” calculates logical disjunction of `a` and `b` with the function [or](/docs/sql-reference/functions/logical-functions#or).

## Logical Negation Operator[вҖӢ](#logical-negation-operator "Direct link to Logical Negation Operator")

Syntax `SELECT NOT a` вҖ” calculates logical negation of `a` with the function [not](/docs/sql-reference/functions/logical-functions#not).

## Conditional Operator[вҖӢ](#conditional-operator "Direct link to Conditional Operator")

`a ? b : c` вҖ“ The `if(a, b, c)` function.

Note:

The conditional operator calculates the values of b and c, then checks whether condition a is met, and then returns the corresponding value. If `b` or `C` is an [arrayJoin()](/docs/sql-reference/functions/array-join) function, each row will be replicated regardless of the "a" condition.

## Conditional Expression[вҖӢ](#conditional-expression "Direct link to Conditional Expression")

```
CASE [x]
    WHEN a THEN b
    [WHEN ... THEN ...]
    [ELSE c]
END

```
