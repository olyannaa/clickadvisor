## check\_conversion\_from\_numbers\_to\_enum[​](#check_conversion_from_numbers_to_enum "Direct link to check_conversion_from_numbers_to_enum")

Throw an exception during Numbers to Enum conversion if the value does not exist in Enum.

Possible values:

- 0 — Disabled.
- 1 — Enabled.

**Example**

```
CREATE TABLE tab (
  val Enum('first' = 1, 'second' = 2, 'third' = 3)
) ENGINE = Memory;

INSERT INTO tab SETTINGS check_conversion_from_numbers_to_enum = 1 VALUES (4); -- returns an error

```

## date\_time\_input\_format[​](#date_time_input_format "Direct link to date_time_input_format")

Allows choosing a parser of the text representation of date and time.

The setting does not apply to [date and time functions](/docs/sql-reference/functions/date-time-functions).

Possible values:

- `'best_effort'` — Enables extended parsing.

ClickHouse can parse the basic `YYYY-MM-DD HH:MM:SS` format and all [ISO 8601](https://en.wikipedia.org/wiki/ISO_8601) date and time formats. For example, `'2018-06-08T01:02:03.000Z'`.
- `'best_effort_us'` — Similar to `best_effort` (see the difference in [parseDateTimeBestEffortUS](/docs/sql-reference/functions/type-conversion-functions#parseDateTimeBestEffortUS)
- `'basic'` — Use basic parser.

ClickHouse can parse only the basic `YYYY-MM-DD HH:MM:SS` or `YYYY-MM-DD` format. For example, `2019-08-20 10:18:56` or `2019-08-20`.

See also:

- [DateTime data type.](/docs/sql-reference/data-types/datetime)
- [Functions for working with dates and times.](/docs/sql-reference/functions/date-time-functions)

## date\_time\_output\_format[​](#date_time_output_format "Direct link to date_time_output_format")

Allows choosing different output formats of the text representation of date and time.

Possible values:

- `simple` \- Simple output format.

ClickHouse output date and time `YYYY-MM-DD hh:mm:ss` format. For example, `2019-08-20 10:18:56`. The calculation is performed according to the data type's time zone (if present) or server time zone.
- `iso` \- ISO output format.
