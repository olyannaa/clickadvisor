## argAndMax[вҖӢ](#argAndMax "Direct link to argAndMax")

Introduced in: v1\.1\.0

Calculates the `arg` and `val` value for a maximum `val` value.
If there are multiple rows with equal `val` being the maximum, which of the associated `arg` and `val` is returned is not deterministic.
Both parts the `arg` and the `max` behave as [aggregate functions](/docs/sql-reference/aggregate-functions), they both [skip `Null`](/docs/sql-reference/aggregate-functions#null-processing) during processing and return not `Null` values if not `Null` values are available.

NoteThe only difference with `argMax` is that `argAndMax` returns both argument and value.

**See also**

- [argMax](/docs/sql-reference/aggregate-functions/reference/argmax)
- [Tuple](/docs/sql-reference/data-types/tuple)

**Syntax**

```
argAndMax(arg, val)

```

**Arguments**

- `arg` вҖ” Argument for which to find the maximum value. [`const String`](/docs/sql-reference/data-types/string)
- `val` вҖ” The maximum value. [`(U)Int8/16/32/64`](/docs/sql-reference/data-types/int-uint) or [`Float*`](/docs/sql-reference/data-types/float) or [`Date`](/docs/sql-reference/data-types/date) or [`DateTime`](/docs/sql-reference/data-types/datetime) or [`Tuple`](/docs/sql-reference/data-types/tuple)

**Returned value**

Returns a tuple containing the `arg` value that corresponds to maximum `val` value and the maximum `val` value. [`Tuple`](/docs/sql-reference/data-types/tuple)

**Examples**

**Basic usage**

```
SELECT argAndMax(user, salary) FROM salary;

```

```

**Extended example with NULL handling**

```
CREATE TABLE test
(
    a Nullable(String),
    b Nullable(Int64)
)
ENGINE = Memory AS
SELECT *
FROM VALUES(('a', 1), ('b', 2), ('c', 2), (NULL, 3), (NULL, NULL), ('d', NULL));

SELECT argMax(a, b), argAndMax(a, b), max(b) FROM test;

```

```
