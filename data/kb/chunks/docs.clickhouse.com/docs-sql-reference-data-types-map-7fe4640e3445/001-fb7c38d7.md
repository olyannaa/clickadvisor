# Map(K, V) \| ClickHouse Docs

- - [Introduction](/docs/sql-reference)- [Data types](/docs/sql-reference/data-types)- Map(K, V)
[Edit this page](https://github.com/ClickHouse/ClickHouse/tree/master/docs/en/sql-reference/data-types/map.md)# Map(K, V)

Data type `Map(K, V)` stores key\-value pairs.

Unlike other databases, maps are not unique in ClickHouse, i.e. a map can contain two elements with the same key.
(The reason for that is that maps are internally implemented as `Array(Tuple(K, V))`.)

You can use use syntax `m[k]` to obtain the value for key `k` in map `m`.
Also, `m[k]` scans the map, i.e. the runtime of the operation is linear in the size of the map.

**Parameters**

- `K` — The type of the Map keys. Arbitrary type except [Nullable](/docs/sql-reference/data-types/nullable) and [LowCardinality](/docs/sql-reference/data-types/lowcardinality) nested with [Nullable](/docs/sql-reference/data-types/nullable) types.
- `V` — The type of the Map values. Arbitrary type.

**Examples**

Create a table with a column of type map:

```
CREATE TABLE tab (m Map(String, UInt64)) ENGINE=Memory;
INSERT INTO tab VALUES ({'key1':1, 'key2':10}), ({'key1':2,'key2':20}), ({'key1':3,'key2':30});

```

To select `key2` values:

```
SELECT m['key2'] FROM tab;

```

```
┌─arrayElement(m, 'key2')─┐
│                      10 │
│                      20 │
│                      30 │
└─────────────────────────┘

```

If the requested key `k` is not contained in the map, `m[k]` returns the value type's default value, e.g. `0` for integer types and `''` for string types.
To check whether a key exists in a map, you can use function [mapContains](/docs/sql-reference/functions/tuple-map-functions#mapContainsKey).

```
CREATE TABLE tab (m Map(String, UInt64)) ENGINE=Memory;
INSERT INTO tab VALUES ({'key1':100}), ({});
SELECT m['key1'] FROM tab;

```

```
┌─arrayElement(m, 'key1')─┐
│                     100 │
│                       0 │
└─────────────────────────┘

```

## Converting Tuple to Map[​](#converting-tuple-to-map "Direct link to Converting Tuple to Map")

Values of type `Tuple()` can be cast to values of type `Map()` using function [CAST](/docs/sql-reference/functions/type-conversion-functions#CAST):

**Example**

```
SELECT CAST(([1, 2, 3], ['Ready', 'Steady', 'Go']), 'Map(UInt8, String)') AS map;

```

```
┌─map───────────────────────────┐
│ {1:'Ready',2:'Steady',3:'Go'} │
└───────────────────────────────┘

```

## Reading subcolumns of Map[​](#reading-subcolumns-of-map "Direct link to Reading subcolumns of Map")

To avoid reading the entire map, you can use subcolumns `keys` and `values` in some cases.

**Example**

```
CREATE TABLE tab (m Map(String, UInt64)) ENGINE = Memory;
INSERT INTO tab VALUES (map('key1', 1, 'key2', 2, 'key3', 3));

SELECT m.keys FROM tab; --   same as mapKeys(m)
SELECT m.values FROM tab; -- same as mapValues(m)

```

```
┌─m.keys─────────────────┐
│ ['key1','key2','key3'] │
└────────────────────────┘

┌─m.values─┐
│ [1,2,3]  │
└──────────┘

```
