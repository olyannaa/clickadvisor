## topK[вҖӢ](#topK "Direct link to topK")

Introduced in: v1\.1\.0

Returns an array of the approximately most frequent values in the specified column. The resulting array is sorted in descending order of approximate frequency of values (not by the values themselves).

Implements the [Filtered Space\-Saving](https://doi.org/10.1016/j.ins.2010.08.024) algorithm for analyzing TopK, based on the reduce\-and\-combine algorithm from [Parallel Space Saving](https://doi.org/10.1016/j.ins.2015.09.003).

This function does not provide a guaranteed result. In certain situations, errors might occur and it might return frequent values that aren't the most frequent values.

**Syntax**

```
topK(N)(column)
topK(N, load_factor)(column)
topK(N, load_factor, 'counts')(column)

```

**Parameters**

- `N` вҖ” The number of elements to return. Default value: 10\. Maximum value of `N = 65536`. [`UInt64`](/docs/sql-reference/data-types/int-uint)
- `load_factor` вҖ” Optional. Defines, how many cells reserved for values. If `uniq(column) > N * load_factor`, result of topK function will be approximate. Default value: 3\. [`UInt64`](/docs/sql-reference/data-types/int-uint)
- `counts` вҖ” Optional. Defines whether the result should contain an approximate count and error value. [`Bool`](/docs/sql-reference/data-types/boolean)

**Arguments**

- `column` вҖ” The name of the column for which to find the most frequent values. [`String`](/docs/sql-reference/data-types/string)

**Returned value**

Returns an array of the approximately most frequent values, sorted in descending order of approximate frequency. [`Array`](/docs/sql-reference/data-types/array)

**Examples**

**Usage example**

```
SELECT topK(3)(AirlineID) AS res
FROM ontime;

```
