# Tuple(T1, T2, ...) \| ClickHouse Docs

- - [Introduction](/docs/sql-reference)- [Data types](/docs/sql-reference/data-types)- Tuple(T1, T2, ...)
[Edit this page](https://github.com/ClickHouse/ClickHouse/tree/master/docs/en/sql-reference/data-types/tuple.md)# Tuple(T1, T2, ...)

A tuple of elements, each having an individual [type](/docs/sql-reference/data-types). Tuple must contain at least one element.

Tuples are used for temporary column grouping. Columns can be grouped when an IN expression is used in a query, and for specifying certain formal parameters of lambda functions. For more information, see the sections [IN operators](/docs/sql-reference/operators/in) and [Higher order functions](/docs/sql-reference/functions/overview#higher-order-functions).

Tuples can be the result of a query. In this case, for text formats other than JSON, values are comma\-separated in `()`. In JSON formats, tuples are output as arrays (in `[]`).

## Creating Tuples[​](#creating-tuples "Direct link to Creating Tuples")

You can use a function to create a tuple:

```
tuple(T1, T2, ...)

```

Example of creating a tuple:

```
SELECT tuple(1, 'a') AS x, toTypeName(x)

```

```
┌─x───────┬─toTypeName(tuple(1, 'a'))─┐
│ (1,'a') │ Tuple(UInt8, String)      │
└─────────┴───────────────────────────┘

```

A Tuple can contain a single element

Example:

```
SELECT tuple('a') AS x;

```

```
┌─x─────┐
│ ('a') │
└───────┘

```

Syntax `(tuple_element1, tuple_element2)` may be used to create a tuple of several elements without calling the `tuple()` function.

Example:

```
SELECT (1, 'a') AS x, (today(), rand(), 'someString') AS y, ('a') AS not_a_tuple;

```

```
┌─x───────┬─y──────────────────────────────────────┬─not_a_tuple─┐
│ (1,'a') │ ('2022-09-21',2006973416,'someString') │ a           │
└─────────┴────────────────────────────────────────┴─────────────┘

```

## Data Type Detection[​](#data-type-detection "Direct link to Data Type Detection")

When creating tuples on the fly, ClickHouse interferes the type of the tuples arguments as the smallest types which can hold the provided argument value. If the value is [NULL](/docs/operations/settings/formats#input_format_null_as_default), the interfered type is [Nullable](/docs/sql-reference/data-types/nullable).

Example of automatic data type detection:

```
SELECT tuple(1, NULL) AS x, toTypeName(x)

```

```
┌─x─────────┬─toTypeName(tuple(1, NULL))──────┐
│ (1, NULL) │ Tuple(UInt8, Nullable(Nothing)) │
└───────────┴─────────────────────────────────┘

```

## Referring to Tuple Elements[​](#referring-to-tuple-elements "Direct link to Referring to Tuple Elements")

Tuple elements can be referred to by name or by index:

```
CREATE TABLE named_tuples (`a` Tuple(s String, i Int64)) ENGINE = Memory;
INSERT INTO named_tuples VALUES (('y', 10)), (('x',-10));

SELECT a.s FROM named_tuples; -- by name
SELECT a.2 FROM named_tuples; -- by index

```

```
┌─a.s─┐
│ y   │
│ x   │
└─────┘

┌─tupleElement(a, 2)─┐
│                 10 │
│                -10 │
└────────────────────┘

```
