```
CREATE TABLE table_with_where
(
    d DateTime,

)
ENGINE = MergeTree
PARTITION BY toYYYYMM(d)
ORDER BY d
TTL d + INTERVAL 1 MONTH DELETE WHERE toDayOfWeek(d) = 1;
```

#### Creating a table, where expired rows are recompressed:[​](#creating-a-table-where-expired-rows-are-recompressed "Direct link to Creating a table, where expired rows are recompressed:")

```
CREATE TABLE table_for_recompression
(
    d DateTime,
    key UInt64,
    value String
) ENGINE MergeTree()
ORDER BY tuple()
PARTITION BY key
TTL d + INTERVAL 1 MONTH RECOMPRESS CODEC(ZSTD(17)), d + INTERVAL 1 YEAR RECOMPRESS CODEC(LZ4HC(10))
SETTINGS min_rows_for_wide_part = 0, min_bytes_for_wide_part = 0;

```

Creating a table, where expired rows are aggregated. In result rows `x` contains the maximum value across the grouped rows, `y` — the minimum value, and `d` — any occasional value from grouped rows.

```
CREATE TABLE table_for_aggregation
(
    d DateTime,
    k1 Int,
    k2 Int,
    x Int,
    y Int
)
ENGINE = MergeTree
ORDER BY (k1, k2)
TTL d + INTERVAL 1 MONTH GROUP BY k1, k2 SET x = max(x), y = min(y);

```

### Removing expired data[​](#mergetree-removing-expired-data "Direct link to Removing expired data")

Data with an expired `TTL` is removed when ClickHouse merges data parts.

When ClickHouse detects that data is expired, it performs an off\-schedule merge. To control the frequency of such merges, you can set `merge_with_ttl_timeout`. If the value is too low, it will perform many off\-schedule merges that may consume a lot of resources.

If you perform the `SELECT` query between merges, you may get expired data. To avoid it, use the [OPTIMIZE](/docs/sql-reference/statements/optimize) query before `SELECT`.

## Disk types[​](#disk-types "Direct link to Disk types")

In addition to local block devices, ClickHouse supports these storage types:

- [`s3` for S3 and MinIO](#table_engine-mergetree-s3)
- [`gcs` for GCS](/docs/integrations/gcs#creating-a-disk)
- [`blob_storage_disk` for Azure Blob Storage](/docs/operations/storing-data#azure-blob-storage)
- [`hdfs` for HDFS](/docs/engines/table-engines/integrations/hdfs)
- [`web` for read\-only from web](/docs/operations/storing-data#web-storage)
- [`cache` for local caching](/docs/operations/storing-data#using-local-cache)
- [`s3_plain` for backups to S3](/docs/operations/backup/disk)
- [`s3_plain_rewritable` for immutable, non\-replicated tables in S3](/docs/operations/storing-data#s3-plain-rewritable-storage)
