### Introduction[‚Äã](#introduction "Direct link to Introduction")

`MergeTree` family table engines can store data on multiple block devices. For example, it can be useful when the data of a certain table are implicitly split into "hot" and "cold". The most recent data is regularly requested but requires only a small amount of space. On the contrary, the fat\-tailed historical data is requested rarely. If several disks are available, the "hot" data may be located on fast disks (for example, NVMe SSDs or in memory), while the "cold" data \- on relatively slow ones (for example, HDD).

This applies to all disk types, including S3 and other object storage disks. For example, you can spread data across multiple S3 buckets within a single volume, or create tiered policies that move data from local disks to S3\. See [Using S3 disks with multiple volumes](#s3-multiple-volumes) for details.

Data part is the minimum movable unit for `MergeTree`\-engine tables. The data belonging to one part are stored on one disk. Data parts can be moved between disks in the background (according to user settings) as well as by means of the [ALTER](/docs/sql-reference/statements/alter/partition) queries.

### Terms[‚Äã](#terms "Direct link to Terms")

- Disk ‚Äî Block device mounted to the filesystem.
- Default disk ‚Äî Disk that stores the path specified in the [path](/docs/operations/server-configuration-parameters/settings#path) server setting.
- Volume ‚Äî Ordered set of equal disks (similar to [JBOD](https://en.wikipedia.org/wiki/Non-RAID_drive_architectures)).
- Storage policy ‚Äî Set of volumes and the rules for moving data between them.

The names given to the described entities can be found in the system tables, [system.storage\_policies](/docs/operations/system-tables/storage_policies) and [system.disks](/docs/operations/system-tables/disks). To apply one of the configured storage policies for a table, use the `storage_policy` setting of `MergeTree`\-engine family tables.
