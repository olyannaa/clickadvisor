- `merge_max_dynamic_subcolumns_in_wide_part` \- a MergeTree setting that limits the number of dynamic subcolumns for each JSON column during merge into a Wide data part.
- `merge_max_dynamic_subcolumns_in_compact_part` \- a MergeTree setting that limits the number of dynamic subcolumns for each JSON column during merge into a Compact data part.
- `max_dynamic_subcolumns_in_json_type_parsing` \- a session setting that limits the number of dynamic subcolumns for each JSON column during parsing of JSON data into a JSON column.

Note: limit on dynamic paths cannot exceed the value specified in `max_dynamic_paths` parameter, even if values of described settings are higher.

## Introspection functions[​](#introspection-functions "Direct link to Introspection functions")

There are several functions that can help to inspect the content of the JSON column:

- [`JSONAllPaths`](/docs/sql-reference/functions/json-functions#JSONAllPaths)
- [`JSONAllPathsWithTypes`](/docs/sql-reference/functions/json-functions#JSONAllPathsWithTypes)
- [`JSONAllValues`](/docs/sql-reference/functions/json-functions#JSONAllValues)
- [`JSONDynamicPaths`](/docs/sql-reference/functions/json-functions#JSONDynamicPaths)
- [`JSONDynamicPathsWithTypes`](/docs/sql-reference/functions/json-functions#JSONDynamicPathsWithTypes)
- [`JSONSharedDataPaths`](/docs/sql-reference/functions/json-functions#JSONSharedDataPaths)
- [`JSONSharedDataPathsWithTypes`](/docs/sql-reference/functions/json-functions#JSONSharedDataPathsWithTypes)
- [`distinctDynamicTypes`](/docs/sql-reference/aggregate-functions/reference/distinctdynamictypes)
- [`distinctJSONPaths and distinctJSONPathsAndTypes`](/docs/sql-reference/aggregate-functions/reference/distinctjsonpaths)

**Examples**

Let's investigate the content of the [GH Archive](https://www.gharchive.org/) dataset for the date `2020-01-01`:

```
SELECT arrayJoin(distinctJSONPaths(json))
FROM s3('s3://clickhouse-public-datasets/gharchive/original/2020-01-01-*.json.gz', JSONAsObject)

```
