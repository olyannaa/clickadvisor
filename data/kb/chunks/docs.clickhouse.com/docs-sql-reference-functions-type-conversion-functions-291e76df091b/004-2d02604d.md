``` accurateCastOrDefault(x, T[, default_value]) ``` **Arguments** - `x` вҖ” A value to convert. [`Any`](/docs/sql-reference/data-types) - `T` вҖ” The target data type name. [`const String`](/docs/sql-reference/data-types/string) - `default_value` вҖ” Optional. Default value to return if conversion fails. [`Any`](/docs/sql-reference/data-types) **Returned value**

Returns the converted value with the target data type, or the default value if conversion is not possible. [`Any`](/docs/sql-reference/data-types)

**Examples**

**Successful conversion**

```
SELECT accurateCastOrDefault(42, 'String')

```

## accurateCastOrNull[вҖӢ](#accurateCastOrNull "Direct link to accurateCastOrNull")

Introduced in: v1\.1\.0

Converts a value to a specified data type.
Like [`accurateCast`](#accurateCast), but returns `NULL` instead of throwing an exception if the conversion cannot be performed accurately.

This function combines the safety of [`accurateCast`](#accurateCast) with graceful error handling.

**Syntax**

```
accurateCastOrNull(x, T)

```

**Arguments**

- `x` вҖ” A value to convert. [`Any`](/docs/sql-reference/data-types)
- `T` вҖ” The target data type name. [`String`](/docs/sql-reference/data-types/string)

**Returned value**

Returns the converted value with the target data type, or `NULL` if conversion is not possible. [`Any`](/docs/sql-reference/data-types)

**Examples**

**Successful conversion**

```
SELECT accurateCastOrNull(42, 'String')

```

```
в”Ңв”ҖaccurateCastOrNull(42, 'String')в”Җв”җ
в”Ӯ 42                               в”Ӯ
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ

```

**Failed conversion returns NULL**

```
SELECT accurateCastOrNull('abc', 'UInt32')

```

```
в”Ңв”ҖaccurateCastOrNull('abc', 'UInt32')в”Җв”җ
в”Ӯ                                бҙәбөҒбҙёбҙё в”Ӯ
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ

```

## formatRow[вҖӢ](#formatRow "Direct link to formatRow")

Introduced in: v20\.7\.0

Converts arbitrary expressions into a string via given format.

NoteIf the format contains a suffix/prefix, it will be written in each row.
Only row\-based formats are supported in this function.

**Syntax**

```
formatRow(format, x, y, ...)

```

**Arguments**

- `format` вҖ” Text format. For example, CSV, TSV. [`String`](/docs/sql-reference/data-types/string)
- `x, y, ...` вҖ” Expressions. [`Any`](/docs/sql-reference/data-types)

**Returned value**

A formatted string. (for text formats it's usually terminated with the new line character). [`String`](/docs/sql-reference/data-types/string)

**Examples**

**Basic usage**

```
SELECT formatRow('CSV', number, 'good')
FROM numbers(3)

```

```
в”Ңв”ҖformatRow('CSV', number, 'good')в”Җв”җ
в”Ӯ 0,"good"
                         в”Ӯ
в”Ӯ 1,"good"
                         в”Ӯ
в”Ӯ 2,"good"
                         в”Ӯ
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ

```

**With custom format**

```
SELECT formatRow('CustomSeparated', number, 'good')
FROM numbers(3)
SETTINGS format_custom_result_before_delimiter='<prefix>\n', format_custom_result_after_delimiter='<suffix>'

```
