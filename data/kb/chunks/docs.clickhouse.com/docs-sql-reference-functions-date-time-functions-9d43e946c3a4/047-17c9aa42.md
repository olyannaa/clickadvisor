[`DateTime`](/docs/sql-reference/data-types/datetime) or [`DateTime64`](/docs/sql-reference/data-types/datetime64) **Examples** **Round down to the start of the minute** ``` SELECT toStartOfMinute(toDateTime('2023-04-21 10:20:30')), toStartOfMinute(toDateTime64('2023-04-21 10:20:30.5300', 8)) FORMAT Vertical ```

```

──────

```

## toStartOfMonth[​](#toStartOfMonth "Direct link to toStartOfMonth")

Introduced in: v1\.1\.0

Rounds down a date or date with time to the first day of the month.

NoteThe return type can be configured by setting [`enable_extended_results_for_datetime_functions`](/docs/operations/settings/settings#enable_extended_results_for_datetime_functions).

**Syntax**

```
toStartOfMonth(value)

```

**Arguments**

- `value` — The date or date with time to round down to the first day of the month. [`Date`](/docs/sql-reference/data-types/date) or [`Date32`](/docs/sql-reference/data-types/date32) or [`DateTime`](/docs/sql-reference/data-types/datetime) or [`DateTime64`](/docs/sql-reference/data-types/datetime64)

**Returned value**

Returns the first day of the month for the given date or date with time. [`Date`](/docs/sql-reference/data-types/date)

**Examples**

**Round down to the first day of the month**

```
SELECT toStartOfMonth(toDateTime('2023-04-21 10:20:30'))

```

```
┌─toStartOfMonth(toDateTime('2023-04-21 10:20:30'))─┐
│                                        2023-04-01 │
└───────────────────────────────────────────────────┘

```

## toStartOfNanosecond[​](#toStartOfNanosecond "Direct link to toStartOfNanosecond")

Introduced in: v22\.6\.0

Rounds down a date with time to the start of the nanoseconds.

**Syntax**

```
toStartOfNanosecond(datetime[, timezone])

```

**Arguments**

- `datetime` — Date and time. [`DateTime64`](/docs/sql-reference/data-types/datetime64)
- `timezone` — Optional. Timezone for the returned value. If not specified, the function uses the timezone of the `value` parameter. [`String`](/docs/sql-reference/data-types/string)

**Returned value**

Input value with nanoseconds. [`DateTime64`](/docs/sql-reference/data-types/datetime64)

**Examples**

**Query without timezone**

```
WITH toDateTime64('2020-01-01 10:20:30.999999999', 9) AS dt64
SELECT toStartOfNanosecond(dt64);

```

```
┌─────toStartOfNanosecond(dt64)─┐
│ 2020-01-01 10:20:30.999999999 │
└───────────────────────────────┘

```

**Query with timezone**

```
WITH toDateTime64('2020-01-01 10:20:30.999999999', 9) AS dt64
SELECT toStartOfNanosecond(dt64, 'Asia/Istanbul');

```

```
┌─toStartOfNanosecond(dt64, 'Asia/Istanbul')─┐
│              2020-01-01 12:20:30.999999999 │
└────────────────────────────────────────────┘

```

## toStartOfQuarter[​](#toStartOfQuarter "Direct link to toStartOfQuarter")

Introduced in: v1\.1\.0

Rounds down a date or date with time to the first day of the quarter. The first day of the quarter is either 1 January, 1 April, 1 July, or 1 October.

NoteThe return type can be configured by setting [`enable_extended_results_for_datetime_functions`](/docs/operations/settings/settings#enable_extended_results_for_datetime_functions).

**Syntax**

```
toStartOfQuarter(value)

```

**Arguments**

- `value` — The date or date with time to round down to the first day of the quarter. [`Date`](/docs/sql-reference/data-types/date) or [`Date32`](/docs/sql-reference/data-types/date32) or [`DateTime`](/docs/sql-reference/data-types/datetime) or [`DateTime64`](/docs/sql-reference/data-types/datetime64)

**Returned value**

Returns the first day of the quarter for the given date or date with time. [`Date`](/docs/sql-reference/data-types/date)

**Examples**

**Round down to the first day of the quarter**

```
SELECT toStartOfQuarter(toDateTime('2023-04-21 10:20:30'))

```
