The first argument can also be specified as [`String`](/docs/sql-reference/data-types/string) in a format supported by [`parseDateTime64BestEffort()`](/docs/sql-reference/functions/type-conversion-functions#parseDateTime64BestEffort). Support for string arguments exists only for reasons of compatibility with MySQL which is expected by certain 3rd party tools. As string argument support may in future be made dependent on new MySQL\-compatibility settings and because string parsing is generally slow, it is recommended to not use it.

**Syntax**

```
toYearWeek(datetime[, mode[, timezone]])

```

**Aliases**: `yearweek`

**Arguments**

- `datetime` вҖ” Date or date with time to get the year and week of. [`Date`](/docs/sql-reference/data-types/date) or [`DateTime`](/docs/sql-reference/data-types/datetime)
- `mode` вҖ” Optional. A mode `0` to `9` determines the first day of the week and the range of the week number. Default `0`. \- `timezone` вҖ” Optional. Time zone. [`String`](/docs/sql-reference/data-types/string)

**Returned value**

Returns year and week number as a combined integer value. [`UInt32`](/docs/sql-reference/data-types/int-uint)

**Examples**

**Get year\-week combinations with different modes**

```
SELECT toDate('2016-12-27') AS date, toYearWeek(date) AS yearWeek0, toYearWeek(date,1) AS yearWeek1, toYearWeek(date,9) AS yearWeek9, toYearWeek(toDate('2022-01-01')) AS prev_yearWeek

```

## today[вҖӢ](#today "Direct link to today")

Introduced in: v1\.1\.0

Returns the current date at moment of query analysis. Same as `toDate(now())`.

**Syntax**

```
today()

```

**Aliases**: `curdate`, `current_date`

**Arguments**

- None.

**Returned value**

Returns the current date [`Date`](/docs/sql-reference/data-types/date)

**Examples**

**Usage example**

```
SELECT today() AS today, curdate() AS curdate, current_date() AS current_date FORMAT Pretty

```

## yesterday[вҖӢ](#yesterday "Direct link to yesterday")

Introduced in: v1\.1\.0

Accepts zero arguments and returns yesterday's date at one of the moments of query analysis.

**Syntax**

```
yesterday()

```

**Arguments**

- None.

**Returned value**

Returns yesterday's date. [`Date`](/docs/sql-reference/data-types/date)

**Examples**

**Get yesterday's date**

```
SELECT yesterday();
SELECT today() - 1;

```
