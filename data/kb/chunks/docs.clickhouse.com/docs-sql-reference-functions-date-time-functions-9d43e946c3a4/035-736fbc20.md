The returned value is in the range 28 to 31\.

**Syntax**

```
toDaysInMonth(datetime)

```

**Arguments**

- `datetime` — Date or date with time to get the number of days in the month from. [`Date`](/docs/sql-reference/data-types/date) or [`Date32`](/docs/sql-reference/data-types/date32) or [`DateTime`](/docs/sql-reference/data-types/datetime) or [`DateTime64`](/docs/sql-reference/data-types/datetime64)

**Returned value**

Returns the number of days in the month of the given date/time. [`UInt8`](/docs/sql-reference/data-types/int-uint)

**Examples**

**Usage example**


## toDaysSinceYearZero[​](#toDaysSinceYearZero "Direct link to toDaysSinceYearZero")

Introduced in: v23\.9\.0

For a given date, returns the number of days which have passed since [1 January 0000](https://en.wikipedia.org/wiki/Year_zero) in the
[proleptic Gregorian calendar defined by ISO 8601](https://en.wikipedia.org/wiki/Gregorian_calendar#Proleptic_Gregorian_calendar).

The calculation is the same as in MySQL's [`TO_DAYS`](https://dev.mysql.com/doc/refman/8.0/en/date-and-time-functions.html#function_to-days) function.

**Syntax**

```
toDaysSinceYearZero(date[, time_zone])

```

**Aliases**: `TO_DAYS`

**Arguments**

- `date` — The date or date with time for which to calculate the number of days since year zero from. [`Date`](/docs/sql-reference/data-types/date) or [`Date32`](/docs/sql-reference/data-types/date32) or [`DateTime`](/docs/sql-reference/data-types/datetime) or [`DateTime64`](/docs/sql-reference/data-types/datetime64)
- `time_zone` — Time zone. [`String`](/docs/sql-reference/data-types/string)

**Returned value**

Returns the number of days passed since date `0000-01-01`. [`UInt32`](/docs/sql-reference/data-types/int-uint)

**Examples**

**Calculate days since year zero**

```
SELECT toDaysSinceYearZero(toDate('2023-09-08'))

```

```
┌─toDaysSinceYearZero(toDate('2023-09-08')))─┐
│                                     713569 │
└────────────────────────────────────────────┘

```

## toHour[​](#toHour "Direct link to toHour")

Introduced in: v1\.1\.0

Returns the hour component (0\-23\) of a `DateTime` or `DateTime64` value.

**Syntax**

```
toHour(datetime)

```

**Aliases**: `HOUR`

**Arguments**

- `datetime` — Date with time to get the hour from. [`DateTime`](/docs/sql-reference/data-types/datetime) or [`DateTime64`](/docs/sql-reference/data-types/datetime64)

**Returned value**

Returns the hour (0\-23\) of `datetime`. [`UInt8`](/docs/sql-reference/data-types/int-uint)

**Examples**

**Usage example**

```
SELECT toHour(toDateTime('2023-04-21 10:20:30'))

```

```
┌─toHour(toDateTime('2023-04-21 10:20:30'))─┐
│                                        10 │
└───────────────────────────────────────────┘

```

## toISOWeek[​](#toISOWeek "Direct link to toISOWeek")

Introduced in: v20\.1\.0

Returns the ISO week number of a date or date with time.

This is a compatibility function that is equivalent to `toWeek(date, 3)`.
ISO weeks start on Monday and the first week of the year contains January 4th.
According to ISO 8601, week numbers are in the range from 1 to 53\.
