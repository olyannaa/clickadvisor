# first\_value \| ClickHouse Docs

- - [Functions](/docs/sql-reference/functions)- [Aggregate functions](/docs/sql-reference/aggregate-functions)- [Aggregate Functions](/docs/sql-reference/aggregate-functions/reference)- first\_value
[Edit this page](https://github.com/ClickHouse/ClickHouse/tree/master/docs/en/sql-reference/aggregate-functions/reference/first_value.md)# first\_value

It is an alias for [`any`](/docs/sql-reference/aggregate-functions/reference/any) but it was introduced for compatibility with [Window Functions](/docs/sql-reference/window-functions), where sometimes it's necessary to process `NULL` values (by default all ClickHouse aggregate functions ignore NULL values).

It supports declaring a modifier to respect nulls (`RESPECT NULLS`), both under [Window Functions](/docs/sql-reference/window-functions) and in normal aggregations.

As with `any`, without Window Functions the result will be random if the source stream is not ordered and the return type
matches the input type (Null is only returned if the input is Nullable or \-OrNull combinator is added).

## examples[​](#examples "Direct link to examples")

```
CREATE TABLE test_data
(
    a Int64,
    b Nullable(Int64)
)
ENGINE = Memory;

INSERT INTO test_data (a, b) VALUES (1,null), (2,3), (4, 5), (6,null);

```

### Example 1[​](#example1 "Direct link to Example 1")

By default, the NULL value is ignored.

```
SELECT first_value(b) FROM test_data;

```

```
┌─any(b)─┐
│      3 │
└────────┘

```

### Example 2[​](#example2 "Direct link to Example 2")

The NULL value is ignored.

```
SELECT first_value(b) ignore nulls FROM test_data

```

```
┌─any(b) IGNORE NULLS ─┐
│                    3 │
└──────────────────────┘

```

### Example 3[​](#example3 "Direct link to Example 3")

The NULL value is accepted.

```
SELECT first_value(b) respect nulls FROM test_data

```

```
┌─any(b) RESPECT NULLS ─┐
│                  ᴺᵁᴸᴸ │
└───────────────────────┘

```

### Example 4[​](#example4 "Direct link to Example 4")

Stabilized result using the sub\-query with `ORDER BY`.

```
SELECT
    first_value_respect_nulls(b),
    first_value(b)
FROM
(
    SELECT *
    FROM test_data
    ORDER BY a ASC
)

```

```
┌─any_respect_nulls(b)─┬─any(b)─┐
│                 ᴺᵁᴸᴸ │      3 │
└──────────────────────┴────────┘

```
