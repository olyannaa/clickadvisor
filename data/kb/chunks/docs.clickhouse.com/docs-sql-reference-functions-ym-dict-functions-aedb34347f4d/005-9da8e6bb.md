### regionToPopulation[​](#regiontopopulation "Direct link to regionToPopulation")

Gets the population for a region. The population can be recorded in files with the geobase. See the section ["Dictionaries"](/docs/sql-reference/statements/create/dictionary/embedded). If the population is not recorded for the region, it returns 0\. In the geobase, the population might be recorded for child regions, but not for parent regions.

**Syntax**

```
regionToPopulation(id[, geobase])

```

**Parameters**

- `id` — Region ID from the geobase. [UInt32](/docs/sql-reference/data-types/int-uint).
- `geobase` — Dictionary key. See [Multiple Geobases](#multiple-geobases). [String](/docs/sql-reference/data-types/string). Optional.

**Returned value**

- Population for the region. [UInt32](/docs/sql-reference/data-types/int-uint).
- 0, if there is none.

**Example**

```
SELECT regionToName(number::UInt32, 'en'), regionToPopulation(number::UInt32) AS id, regionToName(id, 'en') FROM numbers(13);

```
