### Timezones [\#](/blog/working-with-time-series-data-and-functions-ClickHouse#timezones)

Practical cases require having timezones stored as well in many cases. ClickHouse letвҖҷs us set timezone as a last argument to the [DateTime](https://clickhouse.com/docs/en/sql-reference/data-types/datetime) or [DateTime64](https://clickhouse.com/docs/en/sql-reference/data-types/datetime64) types:

```

CREATE TABLE dtz
(
    `id` Int8,
    `t` DateTime('Europe/Berlin')
)
ENGINE = MergeTree
ORDER BY tuple()


```

Having defined a timezone in our DDL, we can now insert times using different timezones:

```

INSERT INTO dtz SELECT 1, toDateTime('2022-12-12 12:13:14', 'America/New_York')
```
