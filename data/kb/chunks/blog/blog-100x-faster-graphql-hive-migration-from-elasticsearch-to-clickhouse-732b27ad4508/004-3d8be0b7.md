with ClickHouse Cloud [\#](/blog/100x-faster-graphql-hive-migration-from-elasticsearch-to-clickhouse#taking-it-to-the-next-level-with-clickhouse-cloud) Our current data pipeline uses a single instance of ClickHouse. It works really well, and so far, we haven’t experienced any hiccups, but the need to scale it to more nodes will eventually arrive.

That’s why we started moving into **ClickHouse Cloud**.

Thanks to its out\-of\-the\-box support for sharding and replication, we can focus on developing new features and let the ClickHouse Cloud take care of the rest.
